# 🎬 Experiência 03: Batalha de Modelos & Engenharia de Prompt (XML)

## 📝 Descrição do Projeto
Este projeto consiste em um estudo comparativo e analítico (benchmarking) entre múltiplos Modelos de Linguagem de Grande Porte (LLMs). O objetivo principal é avaliar a precisão técnica, conformidade e obediência de diferentes arquiteturas de IA frente a um prompt altamente estruturado utilizando marcações XML.

A tarefa envolveu a criação de um prompt estruturado para gerar uma página web *Single Page* em HTML5 e CSS3 interno para uma agência de publicidade focada em mídias e *branding* para jogos de console e computadores. O prompt foi submetido de forma idêntica a sete ferramentas distintas: ChatGPT, Gemini, Claude, Qwen, DeepSeek, Grok e Maritaca. A análise priorizou a fidelidade técnica às restrições e o consumo de tokens em detrimento de critérios puramente estéticos.

## 🚀 Tecnologias Utilizadas
* **Linguagens:** HTML5, CSS3 integrado, Engenharia de Prompt estruturada em XML
* **Modelos de IA Avaliados:** ChatGPT, Gemini, Claude, Qwen, DeepSeek, Grok e Maritaca
* **Métricas de Análise:** Consumo de tokens (verbosidade), completude de seções e fidelidade estrutural às diretrizes

## 📊 Resultados e Aprendizados
O experimento revelou uma disparidade significativa na verbosidade e na capacidade de processamento de instruções complexas entre as ferramentas.

* **Fidelidade Estrutural e Compreensão:** O modelo **Claude** demonstrou a maior compreensão da estrutura XML e entregou o material mais robusto e detalhado sob uma visão completa, atendendo com precisão as diretrizes de design e obrigatoriedades técnicas.
* **Variação de Verbosidade (Tokens):** Houve uma variação extrema no consumo de tokens. Modelos como ChatGPT, Gemini e Maritaca apresentaram respostas mais concisas e básicas (gerando códigos entre 1.100 e 2.600 tokens), enquanto o Claude consumiu entre 6.800 e 7.400 tokens, o que se traduziu diretamente em um código muito mais maduro, criativo e completo (com criação de logo e múltiplos *cases*).
* **Limitações Comuns:** Quase a totalidade dos modelos apresentou dificuldades ou falhas na renderização do mapa com os indicadores de atendimento para a América Latina, limitando-se a caixas simuladas ou estruturas geométricas simples em vez de mapas interativos reais.
* **Metacognição para Prototipagem:** Conclui-se que ferramentas de menor consumo de tokens são ideais para prototipagem rápida e testes de conceito, enquanto modelos altamente detalhados (como o Claude) justificam sua verbosidade ao lidar com lógicas de códigos mais complexos e produções corporativas.

## 🔧 Como Executar
1. Clone este repositório.
2. Acesse a pasta `batalha-modelos-03`.
3. Abra o arquivo `Batalha de Modelos - Experiência 3.pdf` para examinar o quadro comparativo completo e a fundamentação da reflexão crítica.
4. Para visualizar e testar o resultado gerado por cada IA individualmente, abra os respectivos arquivos `.html` diretamente em qualquer navegador web (ex: `claude_pixelforge-agency.html`, `ChatGPT_agencia_de_publicidade_gamer_single_page.html`, etc.).
