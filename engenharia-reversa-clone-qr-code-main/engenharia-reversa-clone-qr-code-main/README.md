# 🎬 Engenharia Reversa e Clonagem de App com IA

## 📝 Descrição do Projeto
Este projeto explora a construção de software assistida por Inteligência Artificial através da engenharia reversa e clonagem de um aplicativo. O foco da atividade não é a codificação manual, mas sim o exercício do pensamento estruturado para guiar a IA na recriação de uma interface e de suas lógicas subjacentes.

O aplicativo clonado resultante deste experimento está hospedado no Google AI Studio e pode ser acessado no link abaixo:
🔗 [Acessar Aplicativo Clonado (AI Studio)](https://ai.studio/apps/74e8c88a-9fb6-4ca3-9636-ccfe7f2c6c52)

## 🚀 Tecnologias Utilizadas
* **Plataforma:** Google AI Studio
* **Técnicas Aplicadas:** Engenharia Reversa de Software, Engenharia de Prompt, Auditoria de Código Gerado por IA

## 📊 Resultados e Aprendizados
A experiência de clonar um aplicativo utilizando ferramentas generativas trouxe reflexões fundamentais sobre o futuro do desenvolvimento e a ética na tecnologia.

**O Novo Papel do Desenvolvedor:**
O uso de IA muda o foco de "saber escrever o código" para "saber resolver o problema". O desenvolvedor júnior deixa de ser um simples digitador e passa a atuar como um gerente de IA. Para não se tornar obsoleto, duas habilidades são indispensáveis:
1. **Pensamento Lógico:** A capacidade de quebrar um aplicativo em passos detalhados para criar instruções precisas (prompts) para a IA.
2. **Senso Crítico de Auditoria:** A habilidade de duvidar da IA, revisando o código gerado para encontrar erros lógicos, falhas de segurança ou problemas de performance.

**Ética e o Limite do Plágio:**
O limite em que a engenharia reversa se torna plágio ocorre quando a cópia do aplicativo é exata e feita com o objetivo de lucrar ou desviar usuários do autor original, sem adicionar valor. 
Como diretriz ética, empresas e desenvolvedores devem adotar a regra de que toda recriação feita por IA exige obrigatoriamente:
* Dar os devidos créditos ao layout original que serviu de inspiração.
* Adicionar pelo menos uma funcionalidade nova ou uma melhoria clara (como otimização de velocidade ou recursos de acessibilidade) que o aplicativo original não possuía.
